# 变更记录策略（Change Policy）

从即日起，**所有项目方向性改动必须记录到 `docs/`**，再进入开发实施。

## 必须记录的改动

- 产品方向与范围变化（新增/删除模块）
- 技术栈变更（框架、语言、数据库、构建链）
- 架构分层调整与边界变化
- 里程碑与交付策略变化

## 最低记录要求

每次方向改动至少包含：
1. 改动背景
2. 决策内容
3. 影响范围
4. 迁移/执行计划
5. 日期与责任人

## 记录位置

- 规则层：`docs/change-policy.md`
- 过程层：`docs/progress-log.md`
- 架构层：`docs/architecture.md`
- 产品层：`docs/prd-v0.1.md`

> 未记录到 docs 的方向改动，视为未生效。
